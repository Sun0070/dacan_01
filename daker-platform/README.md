# DAKER Platform

데이터 경진대회(해커톤) 정보 탐색, 팀 구성, 리더보드 조회를 한 곳에서 제공하는 웹 플랫폼입니다.

> **긴급 인수인계 해커톤** 제출물 — 명세서 기반 바이브 코딩으로 구현 및 확장

---

## 기술 스택

| 구분 | 기술 |
|------|------|
| 프레임워크 | React 18 + TypeScript |
| 빌드 도구 | Vite 5 |
| 스타일링 | Tailwind CSS v3 |
| 라우팅 | React Router v6 |
| 상태 저장 | localStorage (외부 DB 없음) |
| 배포 | Vercel |

## 페이지별 기능

### 홈 (`/`)
- 플랫폼 소개 히어로 섹션
- 해커톤 · 팀 찾기 · 랭킹 바로가기 카드 (실시간 카운트 표시)
- 전체 통계 요약 (해커톤 수 / 모집중 팀 수 / 총 제출 수)
- 이용 방법 3단계 안내
- 주요 해커톤 3개 미리보기 (진행중 → 예정 → 종료 순)

---

### 해커톤 (`/hackathons`)
- **전체 목록**: 11개 해커톤 카드 그리드
- **사이드바 필터**: 상태(전체/진행중/예정/종료), 정렬(최신순/마감임박순), 태그 필터
  - 마감임박순: 종료된 대회는 맨 뒤, 나머지는 마감일 오름차순
- **키워드 검색**: 제목 · 태그 실시간 검색
- **북마크**: 카드별 북마크 버튼, localStorage 저장
- **맞춤 대회 추천 캐러셀** (AI 추천, 70% 이상만 표시)
  - 사용자 역할 · 스킬 기반 룰 기반 점수 산출
  - 카드 컨테이너 외부 좌우 화살표 버튼으로 페이지 이동
  - 헤더에 추천 개수 배지 표시 (예: `4개`)

#### 해커톤 카드 상세
- 상태 배지 (진행중 / 예정 / 종료) + D-day 카운터
- 매칭 % 배지 (추천 섹션에서만 표시)
- **진행률 바**: 진행중 해커톤에서 시작일~마감일 기준 진행 비율 시각화
- **추천 이유 태그** 표시 (추천 섹션에서만)

---

### 해커톤 상세 (`/hackathons/:slug`)
- 탭 구성: 개요 / 평가 / 일정 / 시상 / 팀 / 제출 / 리더보드
- **개요**: 대회 설명, 팀 구성 정책 (솔로 허용 여부, 최대 인원)
- **평가**: 평가 지표 설명, 제출 제한 (일일 횟수, 최대 실행 시간)
  - 투표 기반 대회는 참가자/심사위원 가중치 breakdown 표시
- **일정**: 타임라인 형식의 마일스톤
- **시상**: 순위별 상금 표
- **팀**: Camp 페이지 연동 링크
- **제출**: 제출 가이드, 허용 파일 형식, 제출 항목별 안내
- **리더보드**: 해당 대회 순위표
- **북마크 버튼** + **링크 복사(공유) 버튼**: 헤더 우측

---

### 팀 찾기 (`/camp`)
- **전체 팀 목록**: 26개 팀 카드 그리드
- **사이드바 필터**: 모집 상태(전체/모집중/마감), 해커톤별, 역할별
- **키워드 검색**: 팀 이름 · 소개글 검색
- **팀 만들기 / 수정 / 삭제** (모달 폼)
  - 팀 이름, 해커톤, 소개글, 모집 역할, 인원 수, 연락처 URL 입력
  - 내가 만든 팀에만 수정/삭제 버튼 노출
- **맞춤 팀 추천 캐러셀** (70% 이상만 표시)
  - 역할 일치(+40) · 스킬 매칭(+10) · 소개글 키워드(+5) · 참여 이력(+20) · 모집중(+5) 기반 점수 (60점 만점)
  - 카드 컨테이너 외부 좌우 화살표 버튼으로 페이지 이동
  - 헤더에 추천 팀 수 배지 표시 (예: `10팀`)

#### 팀 카드
- 팀명, 해커톤 이름, 모집 역할 태그
- 역할 매칭 배지: 내 역할과 일치하는 역할 태그에 초록 ✓ 강조
- 매칭 % 배지 (추천 섹션에서만)
- 추천 이유 태그 (추천 섹션에서만)
- 멤버 수 도트 시각화 (5칸 기준)
- 연락하기 버튼 (카카오오픈채팅 / 구글폼 등 외부 링크)

---

### 랭킹 (`/rankings`)
- **전체 리더보드 탭**: 해커톤별 순위표 선택
- **나의 랭킹 탭**: 내가 속한 팀의 순위만 필터링
- 순위 · 팀명 · 점수 · 제출 시각 표시
- 1~3위 금/은/동 메달 아이콘
- 점수 breakdown (투표 대회: 참가자 점수 / 심사위원 점수 분리 표시)
- 제출물 링크 (웹 URL · PDF · 기획서) 버튼

---

### 나의 공간 (`/my`)
- **프로필 편집**: 이름, 역할, 보유 스킬 선택
- **통계 카드 4종**
  - 참여 해커톤 수
  - 내 팀 수
  - 총 제출 수
  - **최고 순위** (전체 리더보드 기준 최고 등수, AwardIcon)
- **내 팀 목록**: 참여 중인 팀 카드 + 수정/탈퇴 기능
- **북마크 해커톤**: 북마크한 대회 카드 목록
- **제출 이력**: 제출한 대회별 이력 타임라인
- 우측 사이드바: 섹션 빠른 이동 내비게이션

## AI 추천 시스템 (룰 기반)

외부 API 없이 사용자 프로필(역할 + 스킬)을 기반으로 점수를 산출합니다.

### 해커톤 추천 (60점 만점)

| 조건 | 점수 |
|------|------|
| 역할 ↔ 해커톤 태그 매칭 (ROLE_TAG_MAP) | +15 |
| 스킬 ↔ 해커톤 태그 매칭 (SKILL_TAG_MAP) | 스킬당 +10 |
| 태그에 스킬명 직접 포함 | 스킬당 +5 |
| 진행중 대회 보너스 | +10 |
| 예정 대회 보너스 | +5 |

**70% 이상(42점↑)인 대회만 추천 캐러셀에 표시**

### 팀 추천 (60점 만점)

| 조건 | 점수 |
|------|------|
| 내 역할이 팀의 lookingFor에 포함 | +40 |
| 내 스킬이 팀의 lookingFor에 포함 | 스킬당 +10 |
| 소개글에 내 스킬 키워드 포함 (최대 2건) | 건당 +5 |
| 내가 참여 중인 해커톤과 동일 대회 | +20 |
| 모집중 여부 | +5 |

**70% 이상(42점↑)인 팀만 추천 캐러셀에 표시**

## 품질 감사 (완성도 체크)

사용성 · 안정성 · 성능 · 접근성/반응형 4개 기준으로 코드 감사를 수행하고 이슈를 수정했습니다.

### 수정된 이슈

| 기준 | 이슈 | 해결 방법 |
|------|------|-----------|
| **사용성** | 전역 검색에서 팀 선택 시 `/camp` 루트로만 이동, 검색어 유실 | React Router `state`로 팀명 전달 → Camp 진입 시 자동 필터 적용 |
| **사용성** | Windows/Linux 사용자에게 Mac 전용 단축키 `⌘K` 표시 | `navigator.userAgent`로 OS 감지 후 `⌘K` / `Ctrl+K` 분기 표시 |
| **사용성** | 마감임박순 정렬 시 이미 종료된 대회가 맨 앞에 표시 | 종료 대회를 맨 뒤로 분리 후 활성 대회만 마감일 오름차순 정렬 |
| **안정성** | `copyLink` 실행 시 Clipboard API 미지원 브라우저에서 무응답 | `.catch()` 에러 처리 추가 + `execCommand` 구형 브라우저 폴백 구현 |
| **접근성** | 전역 검색 `<input>`에 `aria-label` 누락 | `aria-label="해커톤 및 팀 검색"` 추가 |
| **접근성** | 내비게이션 링크에 현재 페이지 표시 누락 | 활성 링크에 `aria-current="page"` 추가 |
| **접근성** | 북마크 버튼의 토글 상태 미전달 | `aria-pressed={isBookmarked}` 추가 |

## 2. 서비스 개요

### 서비스명
**DAKER** (Data + Hacker의 합성어)

### 한줄 소개
데이터 경진대회 정보 탐색부터 팀 구성, 제출, 순위 확인까지 — 해커톤 참가자를 위한 올인원 플랫폼

### 개발 목표
데이터 해커톤 참가자들이 여러 사이트를 오가며 대회 정보를 수집하고, 카카오톡 오픈채팅이나 커뮤니티 게시글로 팀원을 구하는 비효율적인 과정을 단일 플랫폼으로 통합한다. 나아가 사용자 프로필(역할·스킬)을 기반으로 적합한 대회와 팀을 자동 추천하여, 참가 결정부터 팀 합류까지의 마찰을 최소화한다.

### 타겟 사용자

| 유형 | 설명 |
|------|------|
| **입문 참가자** | 처음 해커톤에 참가하려는 ML/데이터 입문자. 어떤 대회가 있는지, 어떤 팀에 합류해야 하는지 막막한 상황 |
| **중급 참가자** | 여러 대회 경험이 있고, 자신의 스킬에 맞는 대회를 빠르게 찾아 좋은 팀원과 조합하고 싶은 사용자 |
| **팀 리더** | 팀을 구성하고 역할별 팀원을 모집하려는 사용자. 모집 공고 작성·관리가 필요 |
| **관람자/응원자** | 리더보드를 통해 결과를 확인하고 싶은 사용자 |

### 기대효과

- **시간 절약**: 대회 탐색 → 팀 합류까지 하나의 서비스에서 완결
- **미스매칭 감소**: 역할·스킬 기반 AI 추천으로 잘 맞는 팀과 대회를 먼저 노출
- **참여율 향상**: 팀 모집 공고와 연락처가 한 곳에 모여 팀 합류 장벽 감소
- **성과 추적**: 리더보드·제출 이력·최고 순위 통계로 자신의 성장 기록 보관

```
1. 첫 방문
   └── localStorage에 daker_seeded_v4 플래그 없음 → 시드 데이터 자동 초기화
       (해커톤 11개, 팀 26개, 리더보드, 제출 이력 등 샘플 데이터 삽입)

2. 나의 공간 (/me) → 프로필 설정
   └── 이름 입력, 역할(ML Engineer 등) 선택, 스킬(Python, PyTorch 등) 선택 → 저장

3. 해커톤 탐색 (/hackathons)
   └── 상단 맞춤 추천 캐러셀에서 70% 이상 매칭 대회 확인
   └── 마음에 드는 대회 북마크 또는 카드 클릭 → 상세 페이지 이동

4. 해커톤 상세 (/hackathons/:slug)
   └── 개요·평가·일정·시상 확인
   └── '팀' 탭 → 팀 찾기 페이지로 이동

5. 팀 찾기 (/camp)
   └── 맞춤 팀 추천 캐러셀에서 역할·스킬 매칭 팀 확인
   └── 팀 카드의 '연락하기' 클릭 → 외부 연락처(카카오오픈채팅 등)로 팀 합류 요청

6. 나의 공간 (/me) → 팀 만들기
   └── 팀 찾기 페이지 '팀 만들기' 버튼 → 폼 작성 → 저장
   └── 나의 공간 '내 팀' 섹션, '참여중인 해커톤' 섹션에 즉시 반영

7. 결과 제출
   └── 해커톤 상세 '제출' 탭 → 기획서/웹링크/PDF 입력 → 제출
   └── 나의 공간 '제출 이력' 섹션에 기록

8. 순위 확인
   └── 랭킹 페이지 → 해커톤 선택 → 전체 리더보드 확인
   └── '나의 랭킹' 탭 → 내 팀 순위만 강조
   └── 나의 공간 통계 카드 '최고 순위' 갱신
```

### 예외 및 오류 처리

| 상황 | 화면 처리 |
|------|-----------|
| 존재하지 않는 해커톤 slug 접근 | `notFound` 상태 활성화 → "해커톤을 찾을 수 없습니다" 안내 메시지 + 목록으로 돌아가기 링크 표시 |
| 프로필 미설정 상태에서 추천 접근 | 추천 캐러셀 자체가 비표시 (role·skills 미설정 시 점수 산출 불가) |
| 팀 만들기 필수 항목 미입력 | Toast 오류 알림("팀 이름과 해커톤을 입력해주세요.") 표시, 저장 차단 |
| 링크 복사 Clipboard API 미지원 | `execCommand` 폴백 자동 실행, 실패 시 Toast 오류 메시지 표시 |
| 필터 결과 없음 | "해당하는 해커톤/팀이 없습니다" 빈 상태 화면 + '필터 초기화' 버튼 표시 |
| 리더보드 데이터 없는 해커톤 | 순위표 영역에 "아직 제출된 결과가 없습니다" 안내 |
| localStorage 읽기/쓰기 실패 | `try-catch`로 조용히 처리, `null` 반환 후 기본값으로 대체 |

### 기술 부채 및 리팩터링 계획
